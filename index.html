Menu Accordion

Problema

O usuário precisa navegar entre as seções principais de uma aplicação enquanto ainda pode navegar rapidamente para a subseção de outra.

Uso

  • Utilize quando quiser os beneficios do menu lateral mas não tiver espaço para listar todas as opções.
  • Use quando tiver mais de duas seções principais, cada uma com duas ou mais subseções.
  • Use quando tiver menos de 10 seções principais.
  • Use apenas quando você tiver dois níveis de navegação principal para mostrar.

Solução

Cada título da seção possui um painel que ao clicar pode ser expandido vertical ou horizontalmente para mostrar as subseções. Os menus verticais são os mais utilizados. A transição entre as opções pode ser feita com uma atualição da página ou uma animação em Javascript. Quando uma seção é clicada ela deve ser expandida enquanto as outras são recolhidas.

Autocomplete

Problema

O usuário precisa de auxilio para reconhecer e se familiarizar com os termos utilizados na aplicação difíceis de lembrar ou com a escrita não utilizada com frequência com facilidade.

Uso

  • Para ajudar em situações de ambigüidade, quando um termo possa estar escrito de diferentes maneiras.
  • Sugerir uma palavra já existente na aplicação por meio da digitação dos primeiras letras que a compõe. Por exemplo quando o usuário não consegue soletrar a palavra "pterodáctilo" mas se lembra das primeiras letras "pte".
  • Aumentar a eficiência/produtividade do usuário no preenchimento de algum formulário por exemplo.
  • Quando é necessário que as informações inseridas por todos usuários permaneçam com um padrão. Usuários podem inserir textos em formatos e escritas diferentes.
  • Quando existir muitos elementos para se usar uma lista selecionável
  • Não usar quando precisar que o usuário tenha uma visão geral de todas as opções disponíveis na aplicação.

Solução

Sugestão de possiveis correspondencias para uma pesquisa no tempo de digitação do usuário. O campo deve ser preditivo quanto mais texto no campo a lista de itens deve ser reduzida para os itens mais relevantes com o texto inserido. A lista deve permitir que o usuário selecione o item desejado utilizando o clique do mouse ou as setas do teclado. Também deve conter um limite padrão de 10 itens no máximo retornados com relevancia no que foi digitado. Pode agrupar itens em categorias.

Navegação em Abas

Problema

O conteúdo precisa ser dividido em seções e acessado usando uma estrutura de navegação simples que fornece uma indicação clara da posição atual.

Uso

  • Quando houver entre 2 a 9 seções de conteúdo que precisem de uma navegação simples.
  • Use quando os títulos das seções forem curtos.
  • Não use quando o conteúdo da seção conter um link de veja mais e o resto do conteúdo ter que ser ocultado.
  • Não use quando precisar mostrar informações especifícos do conteúdo da própria seção, para que não haja confusão em relação ao conteúdo geral da página.

Solução

Utilize uma barra horizontal que contenha as diferentes seções/categorias do site. Cada seção/categoria é representada por uma guia que se parece com um botão. Opcionalmente pode conter uma barra inferior contendo subseções. A guia superior deve se manter estática em todas as páginas às quais as guias se conectam. Deve manter a mesma estrutura entre as guias. Quando utilizar subseções deve deixar claro quais as guias estão ativas no momento.

Modal

Problema

O usuário precisa realizar uma interação até poder continuar visualizar a página atual.

Uso

  • Quando quiser interromper a atividade atual do usuário e ter a total atenção dele para algo importante.
  • Não utilizar para mostrar mensagens de erro, sucesso ou aviso.

Solução

Utilize um modal no qual os usuários não possam interagir com o resto da aplicação enquanto não interagirem ou descartem o modal. O título deve ser correspondente com o botão/link de chamada para aumentar a sensação de familiaridade. Certifique-se que o usuário saiba qual foi a ação que disparou o modal. Deixe visível as opções para que o usuário possa fechar, use métodos que ele já está familiarizado como um botão no canto superior direito "X" ou um botão fechar/cancelar na parte inferior da janela, a tecla "ESC" e o clique fora do modal também é uma boa prática.
